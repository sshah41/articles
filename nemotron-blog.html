Nemotron 3 Super

To understand why a 1-million token context window has been practically impossible until now, we need to abandon abstract descriptions and look at strict linear algebra.

Released by NVIDIA in 2026, Nemotron 3 Super is a 120B-parameter model that redraws the frontier for agentic AI. It achieves this by mathematically dismantling the assumption that the Attention mechanism $A(Q, K, V)$ must be computed at every layer.

Let's map the vector space of the problem.

The KV Tensor Explosion

When feeding standard models (Llama, Standard MoEs) a long prompt, the network computes and stores Key ($K$) and Value ($V$) tensors for every token across all attention layers.

This creates an expanding 4-dimensional tensor $\mathbf{T}_{KV}$ whose memory volume is defined as:

$$ Volume = 2 \cdot L_{attn} \cdot (B \cdot S \cdot N_h \cdot d_h) \cdot p $$

Where $B$ is Batch Size, $S$ is Sequence length, $N_h$ is heads, $d_h$ is dimension, and $p$ is precision bytes (2 for FP16).

Watch the true geometry of the tensor on the right. As the Sequence $S$ approaches 1M tokens, the tensor volume scales linearly until it crashes violently into the 80 GB VRAM Limit of an H100 GPU.

Sparse Attention Tensors

Nemotron 3 Super avoids this mathematical ceiling by drastically reducing the scalar $L_{attn}$.

Watch the tensor collapse in dimension. Rather than calculating Attention at all $L=52$ layers, Nemotron sets $L_{attn} = 6$. These few GQA Layers act as sparse global anchors.

Comparing the volumes, Nemotron's $\mathbf{T}_{KV}$ is roughly 4× smaller. This allows teams to scale the Batch Size $B$ (concurrent agent workflows) without triggering out-of-memory errors.

State-Space Compression ($h_t$)

If only 6 layers handle the $O(N^2)$ scalar dot products of Attention, who processes the sequence $S$?

The core pipeline employs 23 Mamba-2 Layers. As a State-Space Model (SSM), Mamba updates a fixed-size hidden state $h_t$ at each timestep:

$$ h_t = \mathbf{\bar{A}} h_{t-1} + \mathbf{\bar{B}} x_t $$

Because $h_t$ remains constant recursively, Mamba-2 contributes exactly zero volume to the expanding KV tensor, operating in $O(N)$ linear time.

Latent Linear Projections

Interleaved in the stack are 23 LatentMoE Layers.

Standard MoE routes high-dimensional vectors $\mathbf{x} \in \mathbb{R}^d$ directly. LatentMoE explicitly multiplies $\mathbf{x}$ by a projection matrix $W_{down}$ to compress it into a continuous latent bottleneck:

$$ \mathbf{z} = W_{down} \mathbf{x} $$

Observe the geometric plane representing $W_{down}$. The dimensionality collapses into the bottleneck, routes through a subset of 128 micro-experts (only lighting up $k=8$ non-zero activations), and expands back via $W_{up}$.

NVFP4: Precision Quantization

Memory capacity is only half the battle; inference is completely memory-bandwidth bound. To maximize throughput, Nemotron doesn't use standard 16-bit precision natively.

Watch the 16-bit float tensor physically crush itself into a dense block. Nemotron is trained from day one using NVIDIA FP4 (NVFP4), a 4-bit floating point format.

By shrinking the precision scalar $p$ from 2 bytes to 0.5 bytes without post-hoc accuracy loss, the GPU can move matrix weights 4× faster from HBM to SRAM, vastly multiplying execution speed.

The New Constant Baseline

By composing these three innovations — sparse $A(Q,K,V)$, recurrent $\bar{A}h$, and dense 4-bit $W$ matrices — Nemotron 3 Super proves that transformer attention is merely a specialized tool, not a universal requirement.

Codebase-wide generations, legal contract reviews, and multi-agent loops sharing a $S=1,000,000$ context window are no longer abstract theories.

The memory wall was just a solvable equation.
